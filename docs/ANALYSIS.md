# Polymarket 거래봇 객관적 분석

## 거래 주기와 포지션 구조

### 타이밍

| 파라미터 | 값 | 의미 |
|---------|-----|------|
| `EVAL_INTERVAL` | 5초 | 전체 시장 스캔 주기 |
| `MAX_ENTRIES_PER_CYCLE` | 2건 | 한 사이클(5초)에 최대 신규 진입 수 |
| `max_hold_minutes` | 60분 | 포지션 최대 보유 시간 |
| `stale_exit_minutes` | 30분 | 가격 변동 없으면 강제 청산 |

### 동시 포지션

| 파라미터 | 값 |
|---------|-----|
| `max_open_positions` | 5개 |
| condition 중복 차단 | YES/NO 같은 시장에 중복 진입 불가 |
| 실질 동시 보유 | 3~5개 (진입 후 30~60분 내 청산) |

### 예상 거래 빈도

- 오더북 전략: 예측 시장은 얇은 호가창이 많아 imbalance > 0.3 매우 자주 발생
- 모멘텀 전략: 30개+ 데이터포인트 필요 (최소 ~2.5분), 드물게 발생
- 실제 진입: 대부분 오더북 단독 신호, 사이클당 2건 제한
- 예상: 시간당 10~30건 진입, 비슷한 수로 청산

## 승산 분석

### 핵심 문제: 정보 우위(edge)의 부재

| 전략 | 주식시장에서 | 예측시장에서 | 이유 |
|------|-------------|-------------|------|
| 오더북 불균형 | 일부 유효 | 거의 무효 | 예측시장 호가창이 극도로 얇아 노이즈가 대부분. bid 500 vs ask 50은 "방향 압력"이 아니라 한 명이 주문 넣은 것 |
| RSI/BB/EMA | 추세 시장에서 유효 | 거의 무효 | 예측시장 가격은 이벤트 기반으로 움직임. 기술적 "과매도"란 개념이 없음 |
| YES+NO 차익거래 | N/A | 이론적으로 유효, 실전 불가 | 프로 봇들이 밀리초 단위로 선점. 우리는 5초 주기 |

### EV 추정의 순환 논리

```
p_hat = market_price + (max_adjustment * strength)
```

시장가보다 3~8% 더 높다는 추정을 시장 자체의 호가창/기술지표로 도출.
외부 정보(뉴스, 여론조사, 모델)가 없이 시장 데이터만으로 "시장이 틀렸다"고 판단하는 구조.
자기 참조적 추정으로 실제 edge가 0에 수렴.

### 수학적 기대값 시뮬레이션

edge = 0 (무작위 방향 선택) 가정 시:

```
진입 100건 가정:
- 손절 (-5%):       ~35건  ->  35 x (-5%) = -175%
- 익절 (+8%):       ~15건  ->  15 x (+8%) = +120%
- 트레일링 (~+1~3%): ~20건  ->  20 x (+2%) = +40%
- 시간초과/정체:     ~30건  ->  30 x (-1%) = -30%
                                        합계: -45%
```

100건 거래 시 예상 손실: bankroll의 약 -4.5%.
비대칭 청산 규칙 덕에 catastrophic loss는 아니지만 꾸준히 소모.
슬리피지 + 스프레드 비용이 추가되면 실질 손실은 더 큼.

### 경쟁 환경

```
우리 봇:    5초 주기, 룰 베이스, 기술지표, 단일 서버
경쟁자들:   <100ms, ML 모델, 외부 데이터, 전용 인프라, 수십억 자금
```

Polymarket의 상위 마켓메이커/트레이더들은:
- 자체 확률 모델 (베이지안 + 뉴스 NLP)
- 밀리초 단위 실행
- 유동성 공급으로 스프레드 수익
- 방향성 소매 봇의 주문을 역이용

### 종합 평가

| 항목 | 평가 |
|------|------|
| 실전 수익 가능성 | 낮음 (~20% 확률로 단기 운으로 플러스, 장기적으로 마이너스) |
| 리스크 관리 | 양호 (일일 손실 $100 캡, 포지션 5개 제한, 자동 청산) |
| 최대 예상 손실 | 주당 bankroll의 5~15% (설정 기준) |
| 차익거래 수익 | 사실상 0 (5초 주기로는 선점 불가) |

## 실제 edge를 확보하는 방법

1. **외부 확률 모델** - 뉴스/여론조사/소셜 데이터로 `estimated_prob`를 독립 추정
2. **마켓메이킹 전환** - 방향 베팅 대신 양쪽 호가 제공으로 스프레드 수익
3. **더 빠른 실행** - 차익거래는 최소 <1초 실행 필요
4. **백테스트 검증** - 과거 데이터로 전략별 실제 승률 측정 후 운영
